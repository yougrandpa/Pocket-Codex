# Pocket Codex 项目计划

## 1. 项目目标

在手机端（优先 iOS Safari / Web）实现对 Codex 的远程监控与控制，核心能力包括：

- 实时查看 Codex 正在执行的任务和进度。
- 判断任务是否完成、失败、超时或等待输入。
- 从手机端下达新任务或追加指令。
- 支持暂停、继续、取消、重试等任务控制动作。
- 在任务完成或失败时触发通知。

## 2. MVP 范围（第一版）

### 2.1 必做功能

- 任务列表：待执行、运行中、已完成、失败。
- 任务详情：指令内容、开始时间、最后心跳、最近日志、结果摘要。
- 指令下发：创建任务并提交给 Codex 执行。
- 任务控制：暂停、继续、取消、重试。
- 实时状态：通过 WebSocket 或 SSE 推送状态和日志增量。
- 通知能力：先做站内通知，后续扩展系统推送。
- 基础权限：单用户登录和鉴权。

### 2.2 暂不纳入 MVP

- 多租户/多团队权限模型。
- 复杂计费系统。
- 全量原生 iOS 客户端（先用移动 Web，后封装）。

## 3. 产品路线

采用“先 Web，后 iOS”的策略：

1. 先做响应式 Web，确保手机浏览器可用。
2. 稳定后使用 iOS 壳应用（WKWebView）快速上架或内测。
3. 根据使用反馈逐步原生化关键页面（通知、交互、离线体验）。

## 4. 技术方案建议

### 4.1 前端

- Next.js（移动端优先响应式设计）。
- 任务列表 + 详情 + 实时日志流组件化。

### 4.2 后端

- FastAPI 或 NestJS（任选其一，建议先 FastAPI 提升开发速度）。
- 统一任务控制层管理 Codex 会话与状态机。

### 4.3 任务与实时

- Redis + Celery/BullMQ 作为队列与重试机制。
- WebSocket（优先）或 SSE 用于实时状态和日志推送。

### 4.4 数据层

- PostgreSQL 存储任务、状态流转、审计日志。

### 4.5 鉴权与安全

- JWT + Refresh Token。
- 命令白名单、工作目录隔离、最小权限执行。

## 5. 核心架构设计

### 5.1 架构原则

- 手机端不直接运行 Codex，只做控制和展示。
- 服务端统一调度 Codex 进程，保障可观测和可恢复。
- 所有状态变更和控制动作必须可审计。

### 5.2 任务状态机

建议状态流转：

`QUEUED -> RUNNING -> WAITING_INPUT -> SUCCEEDED / FAILED / CANCELED`

补充状态：

- `TIMEOUT`：任务超时。
- `RETRYING`：系统自动重试中。

## 6. API 草案（MVP）

- `POST /tasks`：创建任务（prompt、优先级、工作目录）。
- `GET /tasks`：查询任务列表（支持状态筛选）。
- `GET /tasks/{id}`：查询任务详情（状态、输出、错误、摘要）。
- `POST /tasks/{id}/control`：任务控制（pause/resume/cancel/retry）。
- `POST /tasks/{id}/message`：向运行任务追加指令。
- `GET /stream`：实时事件流（任务状态 + 日志增量）。

## 7. 里程碑（8 周）

### 第 1 周：需求与设计

- 冻结 MVP 范围。
- 完成线框与交互稿。
- 明确状态机与 API 契约。

### 第 2-3 周：后端基础能力

- 搭建任务控制层。
- 接入 Codex 执行器。
- 建立任务、日志、审计相关数据模型。

### 第 4 周：实时能力

- 打通 WebSocket/SSE。
- 完成日志增量推送与断线重连。

### 第 5 周：移动 Web 前端

- 完成任务列表/详情页面。
- 打通创建任务与任务控制动作。

### 第 6 周：稳定性与安全

- 通知能力接入。
- 重试策略、异常恢复、权限加固。

### 第 7 周：灰度与压测

- 并发、恢复、链路稳定性验证。
- 修复核心缺陷。

### 第 8 周：发布

- 生产发布与运行手册。
- 可选：iOS 壳应用封装。

## 8. 验收标准

- 任务状态变化在 2 秒内同步到手机端。
- 任务结束后 10 秒内触发通知。
- 支持 50+ 并发任务，且无任务丢失。
- 失败任务可追溯完整错误日志与执行上下文。
- 移动端主路径（创建 -> 查看 -> 完成）可在 3 分钟内完成。

## 9. 风险与应对

- 子进程异常退出：引入看门狗 + 自动重启。
- 日志过大影响性能：分片存储 + 分页 + 增量拉取。
- 移动网络波动：断线重连 + 序号补偿 + 幂等接口。
- 远程执行安全风险：命令白名单 + 沙箱隔离 + 权限最小化。

## 10. 资源预估

- 1 名全栈：8-10 周上线 MVP。
- 1 后端 + 1 前端：5-7 周上线 MVP。
- 可选 1 iOS：并行提升原生体验，不阻塞 MVP 发布。
